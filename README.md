# Fixje github

This readme contains all the logic we apply in our code version management.


## Rules
- **Repositories** Always lowercase and snakecase like tradein-x-x 
- **Commits** Always with one of these prefixes (fix: , feat: , hotfix: , bug: )
- **Pull requests** Always with one of these prefixes (fix: , feat: , hotfix: , bug: )
- **Branches** Always use *main* for production code
- **Branches** Always use *staging* for customer acceptance on your code
- **Branches** Always use *development* for developer code
- **Branches** Always create new branches from jira ticket so its based on ticket numbers for reference.
